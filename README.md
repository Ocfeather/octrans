# OcTrans — AI 网页翻译插件

一款 Chrome 浏览器扩展,调用 AI 大模型 API(DeepSeek 及其他 OpenAI 兼容接口)翻译网页,并在**原文下方**显示译文。

## 功能特性

- **原文下方对照显示**:译文直接插入到对应段落下面,方便对照阅读。
- **两种翻译模式**:
  - *自动翻译全部* —— 开启后翻译页面中识别到的所有段落。
  - *逐段按钮翻译* —— 每个段落旁出现「译」按钮,点击只翻译该段。
- **广泛的内容识别**:基于文本节点 + 块级祖先分组,能识别 `<div>` / `<span>` 等非语义标签承载的正文,而不仅是 `<p>`。
- **跳过已是目标语言的内容**:例如目标为中文时,自动跳过本就是中文的段落。
- **图片文字翻译(OCR)**:启用后,悬停图片出现「译图」按钮,点击调用视觉模型识别并翻译图片中的文字,以半透明遮罩覆盖在图片上显示。
- **模型可选可输**:填入 API Key 后点 `↻` 从接口的 `/models` 拉取模型列表;下拉选择,也可直接手动输入(适配未提供 `/models` 的厂商)。
- **多厂商通用**:任何 OpenAI 兼容接口均可使用(DeepSeek、OpenAI、Moonshot、通义千问、OpenRouter,以及 Ollama / LM Studio 等本地服务)。
- **动态页面支持**:通过 MutationObserver 处理无限滚动 / 动态加载的新内容。

## 安装

> 暂未上架 Chrome 网上应用店,目前通过「加载已解压的扩展程序」方式安装。

1. 前往 [Releases 页面](https://github.com/Ocfeather/octrans/releases/latest) 下载安装包 **`octrans.zip`**(或[点此直接下载最新版](https://github.com/Ocfeather/octrans/releases/latest/download/octrans.zip))。
2. 将 `octrans.zip` **解压**到任意文件夹(解压后文件夹内应能直接看到 `manifest.json`)。
3. 打开 Chrome,地址栏访问 `chrome://extensions`。
4. 打开右上角的**开发者模式**。
5. 点击**加载已解压的扩展程序**,选择第 2 步解压出的文件夹。

> 想跟进最新开发版,也可克隆源码后直接加载该文件夹:
> ```bash
> git clone https://github.com/Ocfeather/octrans.git
> ```

## 配置

点击浏览器工具栏中的扩展图标,打开设置弹窗:

| 字段 | 说明 | 默认值 |
| --- | --- | --- |
| API 地址 | OpenAI 兼容的对话端点 | `https://api.deepseek.com/chat/completions` |
| API Key | 你的密钥(仅保存在本地 `chrome.storage`) | 空 |
| 模型(文本翻译) | 点 `↻` 获取列表后下拉选择,或手动输入 | `deepseek-chat` |
| 视觉 API 地址 | 图片翻译专用端点,**留空则与文本相同** | 空 |
| 视觉 API Key | 图片翻译专用密钥,**留空则与文本相同** | 空 |
| 图片翻译模型(视觉模型) | 用于 OCR 的视觉模型(如 `glm-5v-turbo`、`gpt-4o`、`qwen-vl` 等),需手动选择/输入 | 空 |
| 启用图片翻译 | 是否在图片上显示「译图」按钮 | 关闭 |
| 翻译模式 | 自动翻译全部 / 逐段按钮翻译 | 自动翻译全部 |
| 目标语言 | 中文 / English / 日本語 / 한국어 等 | 中文 |

> 图片翻译需选用**支持图像输入的视觉模型**,文本模型(如 `deepseek-chat`)无法识别图片。

填写后点击**保存设置**。

## 使用

**网页文字翻译:**

1. 在弹窗中点击**开启翻译**。
2. *自动模式*:页面段落下方陆续出现译文。
3. *按钮模式*:点击段落旁的「译」按钮翻译该段。
4. 再次点击**关闭翻译**可移除全部译文,恢复原文。

> 切换翻译模式后,需先关闭再开启翻译(或刷新页面)使其生效。

**图片文字翻译:**

1. 在弹窗中配置好**图片翻译模型**并勾选**启用图片翻译**。
2. 将鼠标悬停到图片上,左上角出现「译图」按钮,点击即可。
3. 译文以半透明遮罩覆盖在图片上,点击遮罩右上角 `×` 关闭。

## 工作原理

- `manifest.json` —— Manifest V3 配置。
- `content.js` —— 遍历可见文本节点,按块级祖先分组,在原文下注入译文 / 按钮。
- `background.js` —— Service Worker,调用 AI API。请求使用索引键对象(`{"0":…,"1":…}`)以保证译文与原文按序对应,单条缺失不影响整批。
- `popup.*` —— 设置界面。

## 隐私说明

API Key 仅保存在浏览器本地的 `chrome.storage`,不会上传到除你所配置的 AI 接口之外的任何服务器。
